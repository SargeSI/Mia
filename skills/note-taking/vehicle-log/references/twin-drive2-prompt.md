# Промт для близняшки: черновик поста Drive2 по бортжурналу

Ниже шаблон промта для Cursor-Мии (близняшки), когда нужно написать черновик поста на основе данных из LightRAG.

## Шаблон

```
Напиши черновик поста для Drive2 — [ТЕМА ПОСТА].

ИСТОЧНИК ДАННЫХ:
Сделай НЕСКОЛЬКО отдельных rag_query в LightRAG, чтобы покрыть все нужные записи. Не ограничивайся одним запросом — один запрос возвращает только часть фактов.

Запрос 1: [ключевые слова из записи 1]
Запрос 2: [ключевые слова из записи 2]
Запрос 3: [ключевые слова из записи 3 — если есть]

Из каждого запроса возьми ВСЕ факты. Источники — файлы [перечислить имена].

КРИТИЧЕСКОЕ ПРАВИЛО:
Используй ТОЛЬКО факты из бортжурнала. Если факта нет — НЕ выдумывай. Не придумывай названия цветов, опции, комплектации, детали, которых нет в данных. Лучше пробел, чем ложь.

СТРУКТУРА ПОСТА:
1. [Раздел 1]
2. [Раздел 2]
...

СТИЛЬ:
- От первого лица (Сергей)
- Техническая глубина, инженерный подход, без воды
- Живой язык, не сухой отчёт
- Можно с юмором, но без перегибов

ФОТО:
В места, где нужны фото, вставляй [ФОТО: описание того, что должно быть на фото].

ХЭШТЕГИ: #[теги]
```

## Пример для E53 (покупка)

```
Напиши черновик поста для Drive2 — история покупки BMW E53 X5 4.4i N62 (VIN WBAFB51040LU50083).

ИСТОЧНИК ДАННЫХ:
Сделай ТРИ отдельных rag_query в LightRAG:

Запрос 1: «E53 покупка 600000 рублей ноябрь 2023 заправка снег эвакуатор Воронеж кандидат документы»
Запрос 2: «E53 внешний осмотр салон замасленный руль облезлый багажник бежевый антидым бутылка»
Запрос 3: «E53 подъёмник тормозные суппорты диски ржавые рейка балка фаркопа ABS вентилятор»

Из каждого запроса возьми ВСЕ факты. Источник — файлы vehicle-log-e53-001-purchase.md, vehicle-log-e53-002-exterior.md, vehicle-log-e53-003-lift.md.

КРИТИЧЕСКОЕ ПРАВИЛО:
Используй ТОЛЬКО факты из этих трёх файлов. Если факта нет — НЕ выдумывай. Никаких «Oxford-grün», «Dakota pastell-grün», «пневмоподвеска», «целый кузов без коррозии». Лучше пробел, чем ложь.

СТРУКТУРА ПОСТА:
1. Предыстория — почему начали искать X5 (E39 жены встала на кузовной ремонт)
2. Требования к поиску (E53 рест, V8, подогрев руля, не чёрная)
3. Три кандидата (Владимир, Воронеж, Москва) — с конкретными проблемами каждого
4. Осмотр в -20 на заправке — как смотрели, что увидели, эвакуатор
5. Первый внешний осмотр после покупки — мотор, ГУР, кузов, салон (включая бежевые обшивки, бутылку антидыма)
6. Осмотр на подъёмнике — ВСЕ пункты: тормоза (суппорты+диски+колодки по кругу), рейка (сломана, но собрали из двух), ступичные подшипники, балка фаркопа вместо амортизаторов, течи приводов, ABS, xDrive, вентилятор, фара
7. Итог: что купили за 600 тысяч — живой мотор без задиров vs всё остальное

СТИЛЬ:
- От первого лица (Сергей)
- Техническая глубина, инженерный подход, без воды
- Живой язык, не сухой отчёт
- Можно с юмором

ФОТО:
Вставляй [ФОТО: описание]. Обязательные: дырка в полу E39, X5 на заправке в снегу, эвакуатор, салон/руль/обшивки, подъёмник/балка фаркопа/рейка/тормоза.

ХЭШТЕГИ: #BMW #X5 #E53 #N62 #Бортжурнал #Покупка
```

## Почему промт именно такой

1. **Три запроса, а не один** — LightRAG возвращает только релевантные чанки. Один запрос не покрывает все три записи бортжурнала. Нужно по запросу на каждую часть.

2. **Ключевые слова из содержимого, не имя файла** — LightRAG не индексирует имена файлов для семантического поиска. Запрос «vehicle-log-e53-v2» не найдёт документ. Нужны слова из текста: «600000», «ноябрь», «Воронеж», «снег».

3. **Явный запрет на выдумывание** — если бортжурнал не найден, LLM склонна сочинять факты из общей информации о машине. Прямой запрет с примерами снижает вероятность галлюцинаций.

4. **Подробная структура** — без неё близняшка пропускает целые разделы (как в первой попытке: суппорты, балка фаркопа, ABS, вентилятор были пропущены).
