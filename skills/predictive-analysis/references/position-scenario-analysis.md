# Position Scenario Analysis — Методика анализа позиций

Расширение predictive-analysis для применения к конкретным торговым позициям. Когда Сергей спрашивает «какой вариант лучше» или просит обсчитать сценарии для позиции.

## Принципы

1. **Не рекомендация.** Аналитик считает, трейдер решает. Вывод: «при X вероятность убытка Y», не «делай Z».
2. **Мультивариантный расчёт.** Всегда сравнивать минимум 3 варианта действий.
3. **Два критерия:** E[P&L] (матожидание) и худший исход (риск-менеджмент).
4. **Позиция ≠ прогноз.** Прогноз говорит куда пойдёт цена. Позиция добавляет: цена входа, экспирация, размер, корреляции с другими позициями.

## Протокол

### Шаг 1. Описать позицию
- Инструмент, контракт, экспирация
- Цена входа, текущая цена, размер позиции
- Коррелирующие позиции (другие инструменты того же портфеля)
- Ключевой вопрос: «хочу закрыть/сократить/докупить/хеджировать?»

### Шаг 2. Загрузить сценарии из predictive-analysis
- Взять готовые гипотезы (H1/H2/H3) и вероятности из ACH
- При необходимости детализировать: добавить под-исходы (gap up внутри срыва, паника внутри дна)

### Шаг 3. Сформулировать варианты действий
Каждый вариант — полная траектория с момента «сейчас» до целевого горизонта:
- Что делаем сейчас (закрыть X%, открыть шорт, ждать, etc.)
- Что делаем при наступлении события (дно, gap up, плавный рост)
- Что делаем к целевому горизонту

### Шаг 4. Посчитать P&L для каждого варианта × исхода
Для каждого варианта:
- Закрытая часть: фиксируемый P&L = кол-во × (цена_закрытия − цена_входа)
- Оставленная часть: P&L к экспирации/горизонту
- Новые позиции: P&L от входа до горизонта

### Шаг 5. Рассчитать E[P&L] и худший исход
- E[P&L] = Σ(P_исхода × P&L_исхода)
- Худший исход: минимальный P&L по всем исходам, его вероятность
- Риск забора (остаться вне рынка при росте): вероятность × убыток

### Шаг 6. Выдать сравнительную таблицу
| Вариант | E[P&L] | Худший исход | P худшего | Убыток в худшем |
|----------|:---:|------|:---:|:---:|
| ... | | | | |

Не ранжировать. Предоставить данные. Отдельно выделить ключевой конфликт (trade-off: E vs защита от хвостового риска).

## Пример структуры расчёта

```
Исходы:
  Gap up ($10):    P=6%,  дно=None,  июль_выход=$93,  дек_вход=$95,  дек_выход=$100
  Плавный срыв:    P=14%, дно=None,  июль_выход=$88,  дек_вход=$88,  дек_выход=$100
  Дно $70-73:      P=35%, дно=$71,   июль_выход=$90,  дек_вход=$71,  дек_выход=$90
  Промеж. $74-79:  P=27%, дно=$76,   июль_выход=$88,  дек_вход=$76,  дек_выход=$88
  Паника $65-69:   P=10%, дно=$67,   июль_выход=$85,  дек_вход=$67,  дек_выход=$85
  Дно + срыв:      P=8%,  дно=$70,   июль_выход=$100, дек_вход=$70,  дек_выход=$100

Варианты:
  1. Кэш + ждать дна: закрыть 100% сейчас (−$12), на дне лонг дек
  2. Шорт ближний: закрыть лонг (−$12), шорт июль, на дне переворот в лонг дек
  3. Держать всё: держать июль, на дне докупить дек
  4. Сократить X%: закрыть X% сейчас, остальное держать, на дне докупить дек
```

## Связь с predictive-analysis

Этот документ — extension к `predictive-analysis`. Использует:
- ACH-сценарии и вероятности (Шаг 3 основного скилла)
- Admiralty-веса для оценки качества прогнозов
- Kent-шкалу для формулировки уверенности в сценариях
- Горизонты и milestones для привязки к экспирациям

Загружать `predictive-analysis` первым, затем применять этот reference для привязки к позиции.
