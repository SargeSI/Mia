# Модели атрибуции результативности

Соответствует **Фаза 0, F0.9** в [../PREDICTIVE.md](../PREDICTIVE.md).

---

## 1. Суть

Разложение **доходности портфеля** (или разницы с бенчмарком) на **компоненты**: вклад аллокации (распределение по секторам/странам/классам активов), вклад выбора бумаг (stock picking), вклад тайминга, вклад валюты и т.д. Ответ на вопрос: «за счёт чего портфель обыграл или проиграл бенчмарк?»

---

## 2. Разновидности

| Метод | Описание |
|-------|----------|
| **Brinson–Fachler (Brinson–Hood–Beebower)** | Разложение избыточной доходности на **allocation effect** (разница весов портфеля и бенчмарка × доходность бенчмарка по сегменту), **selection effect** (разница доходностей при весах бенчмарка), **interaction**. Используются веса и доходности по сегментам (сектор, страна, класс актива). |
| **Факторная атрибуция** | Разложение по факторам (рынок, размер, стоимость, моментум и т.д.): вклад = экспозиция портфеля к фактору × доходность фактора за период. Требует факторную модель (F0.1). |
| **Fixed Income Attribution** | Для облигаций: разложение на **duration/carry**, **curve** (сдвиг кривой), **spread**, **currency**. |

---

## 3. Происхождение и авторы

- **Brinson:** Brinson, Hood, Beebower (1986) «Determinants of Portfolio Performance», Financial Analysts Journal; Brinson, Fachler (1985) — сегментная атрибуция.
- **Факторная атрибуция:** опирается на Fama–French, Carhart, макрофакторные модели; провайдеры — Barra, Axioma, Northfield.
- **Fixed income:** надстройки над duration и кривой доходности; стандарты — CFA Institute (GIPS, Performance Attribution).

---

## 4. Где применялась и применяется сейчас

- **Управляющие компании:** отчётность перед клиентами; оценка вклада аллокации vs выбора.
- **Пенсионные фонды, страховщики:** анализ эффективности внешних и внутренних управляющих.
- **Сейчас:** стандартный блок в Aladdin (BlackRock), SimCorp, FactSet, Bloomberg PORT; отчёты по GIPS.

---

## 5. Литература и регламентация

- **Стандарты:** CFA Institute «Global Investment Performance Standards (GIPS)»; «Performance Attribution» (CFA curriculum).
- **Учебники:** Bacon «Practical Portfolio Performance Measurement and Attribution»; колодки Brinson в статьях и гайдах провайдеров.

---

## 6. Данные, требуемые в БД

| Данные | Назначение |
|--------|------------|
| **Доходности активов** за период | По каждому инструменту портфеля и бенчмарка. |
| **Веса портфеля** по сегментам (сектор, страна) и по инструментам | Allocation: сравнение с весами бенчмарка. |
| **Веса бенчмарка** по тем же сегментам | База для Brinson. |
| **Доходности бенчмарка** по сегментам | Доходность сегмента бенчмарка для allocation effect. |
| **Факторные экспозиции и доходности факторов** (для факторной атрибуции) | Из факторной модели (F0.1). |
| **Справочник** (сектор, страна по инструменту) | Группировка в сегменты. |

Периодичность: обычно месячная или квартальная для атрибуции.

---

## 7. Какие данные уже есть в Aladdin

| Требование | Таблица/источник | Наличие |
|------------|-------------------|---------|
| Доходности активов | **fact_quote** (date, instrument_id, close/adjusted_close) | Есть. |
| Веса портфеля по инструментам | **fact_position_snapshot** или агрегат **fact_trade** | Есть. |
| Сектор, страна по инструменту | **dim_instrument** (sector, country) | Есть (semantic_layer.md). |
| Бенчмарк (инструмент-индекс) | fact_quote для бенчмарка; веса бенчмарка по сегментам | Котировки бенчмарка — есть при загрузке индекса. **Веса бенчмарка по секторам/странам** — не хранятся в БД по умолчанию; нужны внешние данные (состав индекса) или конфиг. |
| Доходности факторов (SMB, HML и т.д.) | Для факторной атрибуции | Как в 01_factor_models: при наличии загрузки факторных рядов — возможно. |

**Итог:** для **Brinson по сектору/стране** в Aladdin достаточно fact_quote, позиций и dim_instrument (sector, country). Веса бенчмарка по сегментам нужно задавать (конфиг или загрузка состава индекса). Для **факторной атрибуции** дополнительно нужны факторные ряды и рассчитанные экспозиции (F0.1). Модуль атрибуции — перспектива (дашборд «Доходность и атрибуция», INTERFACE.md).
