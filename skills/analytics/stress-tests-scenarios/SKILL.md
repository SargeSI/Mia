# Стресс-тесты и сценарный анализ портфеля

Соответствует **Фаза 0, F0.3** в [../PREDICTIVE.md](../PREDICTIVE.md).

---

## 1. Суть модели

Задаётся **неблагоприятный сценарий** — набор шоков по факторам (например «рынок −20%», «нефть +30%», «ставка +2 п.п.»). Модель переоценивает позиции при этих шоках и выдаёт **оценку потерь** (в деньгах или в %). Основа: факторные экспозиции (беты) или переоценка по правилам (duration для облигаций, дельта для опционов).

---

## 2. Типы сценариев

| Тип | Описание | Пример |
|-----|----------|--------|
| **Исторический** | Повторение условий прошлого кризиса | «Как в 2008», «COVID март 2020». |
| **Гипотетический (what-if)** | Произвольные сдвиги факторов | «Нефть +30%, ставка +2 п.п.». |
| **Обратный (reverse stress)** | «При каком шоке потери достигнут X%?» | «Какой сдвиг нефти даст −10% портфеля?». |

Регуляторные стресс-тесты (ФРС, EBA, ЦБ) задают сценарии в виде макропеременных на 2–3 года вперёд; банки пересчитывают P&L и капитал.

---

## 3. Происхождение и авторы

- **Практика:** банковский стресс-тест — с 1990-х (например, скандинавский кризис); массовое внедрение после 2008 (ФРС CCAR, EBA EU-wide stress tests).
- **Теоретическая база:** сценарный анализ в финансах опирается на факторные модели (бета к факторам) и на модели ценообразования (duration, Greeks). Литература: регламенты Basel, документы ФРС/EBA; учебники по риск-менеджменту (Hull, Jorion).

---

## 4. Где применялась и применяется сейчас

- **Банки:** обязательные стресс-тесты по сценариям регулятора (ФРС, EBA, Банк России) и внутренние сценарии.
- **Управление активами:** BlackRock Aladdin, SimCorp, внутренние системы — сценарный анализ портфеля по факторам (рынок, ставка, сырьё, курс).
- **Пенсионные фонды, страховщики:** ALM и Solvency II — сценарии для активов и обязательств.

---

## 5. Литература и регламентация

- **Регуляторы:** Basel III (стресс-тестирование рыночного и кредитного риска); ФРС CCAR; EBA Guidelines on stress testing; Банк России (положения о стресс-тестах).
- **Учебники:** Jorion «Value at Risk», Hull «Risk Management»; «Stress Testing» (Fed, ECB публикации).

---

## 6. Данные, требуемые в БД

| Данные | Назначение |
|--------|------------|
| **Параметры сценария** | Набор сдвигов факторов (название, описание, JSON: factor_id → shock). |
| **Текущие цены/ставки** | База для применения шока (close из fact_quote, value из fact_macro). |
| **Позиции портфеля** | Текущие веса/количество по инструментам. |
| **Факторные экспозиции (беты)** | Чувствительность портфеля (или каждого инструмента) к каждому фактору — из факторной модели (F0.1). |
| **Справочник сценариев** | id, name, description, parameters_json. |

---

## 7. Какие данные уже есть в Aladdin

| Требование | Таблица/источник | Наличие |
|------------|-------------------|---------|
| Параметры сценария, справочник | **dim_scenario** (id, name, description, parameters_json, created_at) | Есть (semantic_layer.md). |
| Текущие цены | **fact_quote** (последняя close по instrument_id), **fact_macro** (последнее value по indicator_id, country_id) | Есть при настроенных котировках и макро. |
| Позиции | **fact_position_snapshot** или агрегат из **fact_trade** | Есть (см. 02_portfolio_risk_metrics). |
| Беты к факторам | Результат расчёта по факторной модели (F0.1); хранятся в кэше/таблице или пересчитываются на лету | В БД готовой таблицы бет по умолчанию нет — их даёт модуль факторного анализа по fact_quote/fact_macro. |
| Результаты прогонов | **fact_scenario_result** (scenario_id, run_date, instrument_id, initial_value, stressed_value, loss_pct) | Есть в схеме (semantic_layer.md). |

**Итог:** структура данных для сценарного анализа в Aladdin **есть**: dim_scenario, fact_quote, fact_macro, позиции, fact_scenario_result. Для работы ScenarioEngine нужна реализация модуля, берущего беты (из расчёта по F0.1 или из кэша) и применяющего parameters_json к позициям и записывающего в fact_scenario_result.
