# Тестирование: VANOS и проблема single-vector

**Дата:** 2026-05-29
**Контекст:** проверка recall router после интеграции в gateway

## Тестовый запрос

«ванос е39» — ожидалось: BMW E39 сессия (cafbb586...)

## Результаты pgvector (search_sessions.py)

| Сессия | sim | Порог 0.5 |
|---|---|---|
| Кулинария | 0.5468 | ✅ ложное срабатывание |
| Born_Mia | 0.5084 | ✅ |
| BMW E39 | 0.4337 | ❌ не найден |

## Причина

Embedding BMW-сессии создан по тексту «BMW E39: диагностика рулевого». В embedding нет семантики VANOS — слово не встречалось. Когда запрос «ванос е39», модель qwen3-embedding не находит связи между «ванос» и «рулевое».

При этом в Obsidian (LightRAG) информация о VANOS есть:
- `ванос-стук-n63.md` — полный план ремонта
- `BMW E39 535i.md` — основная заметка

LightRAG нашёл: адаптеры, обратные клапаны 11111734667, маслонасос 11411438280, датчик давления 12612367061, масло 10W-60.

## Выводы

1. **Single-vector на сессию — нерабочее решение.** Если тема эволюционировала (от рулевого к VANOS), embedding не отражает новое содержание.
2. **LightRAG компенсирует** — нашёл VANOS когда pgvector не справился.
3. **Решение:** кластерная архитектура (session_clusters) — каждый кластер подтемы получает свой embedding.
