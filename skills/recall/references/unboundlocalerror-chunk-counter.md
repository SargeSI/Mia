# UnboundLocalError: глобальный счётчик в gateway/run.py

**Дата:** 30.05.2026 09:20-09:30
**Исполнитель:** Cursor-Мия (близняшка)
**Вердикт:** исправлено

## Инцидент

Бот упал с `UnboundLocalError: cannot access _chunk_msg_counter` после добавления cross-session routing.

## Корневая причина

`_chunk_msg_counter` — модульная переменная (стр. 159 gateway/run.py). Используется внутри двух разных функций `run_sync()`:

- Строка 12003: в `_run_agent_via_proxy()` — агент для прокси
- Строка 16701: в `_run_agent()` — основной agent loop

Python требует `global _chunk_msg_counter` внутри **каждой** функции, где происходит присваивание (`+= 1`). Если global объявлен только в одной — вторая падает.

## Хронология

1. **Итерация 1 (09:20):** добавлен `global` в `_run_agent_via_proxy()` (строка 12003, отступ 16). Бот перезапущен. Та же ошибка.
2. **Итерация 2 (09:30):** добавлен `global` в `_run_agent()` (строка 16702, отступ 12). Именно эта функция реально вызывалась. Бот перезапущен — ошибок нет.

## Урок

Если модульная переменная используется в нескольких вложенных функциях — `global` нужен в **каждой** из них. Проверять grep'ом все точки использования, а не только первую найденную.

```bash
grep -n "_chunk_msg_counter" gateway/run.py  # найти все использования
# → проверить, что перед каждым `+= 1` есть `global _chunk_msg_counter`
```
