# Admiralty Code (NATO System) — оценка надёжности источника и достоверности информации

Подробное описание методики оценки разведывательной информации, используемой в NATO и странах Five Eyes. Связь с предиктивной аналитикой Aladdin: **[../PREDICTIVE.md](../PREDICTIVE.md)**, раздел 10.1.

---

## 1. Что это такое

**Admiralty Code** (также **Admiralty System**, **NATO System**) — система двухкомпонентной оценки разведывательных данных:

1. **Надёжность источника (Source Reliability)** — буквенный код A–F: насколько источник в прошлом оказывался достоверным и компетентным.
2. **Достоверность информации (Information Credibility / Accuracy)** — цифровой код 1–6: насколько данное конкретное сообщение подтверждено и непротиворечиво.

Итоговая оценка записывается как **двухсимвольный код** (например **A1**, **B2**, **F6**). Ключевой принцип: **источник и информация оцениваются раздельно**. Надёжный источник может передать ложные сведения (дезинформация, ошибка); ненадёжный — случайно дать верную информацию. Поэтому один рейтинг «на всё» не допускается.

---

## 2. Откуда взялась и кем придумана

- **Происхождение:** система разработана **Королевским военно-морским флотом Великобритании (Royal Navy)** во время **Второй мировой войны** для оценки поступающей разведки.
- **Авторство:** конкретное имя создателя в открытых источниках не фигурирует; это ведомственный стандарт британской разведки.
- **Стандартизация:** впоследствии принята **NATO** и странами **Five Eyes** (США, Великобритания, Канада, Австралия, Новая Зеландия) как общепринятый способ передачи уровня уверенности в разведданных между союзниками и между подразделениями.

---

## 3. Где используется

| Область | Применение |
|---------|------------|
| **Военная разведка** | NATO, армии стран-союзников (в т.ч. полевые уставы: US Army FM 2-22.3, UK Joint Warfare Publication 2-00) |
| **Стратегическая разведка** | Оценка сообщений о намерениях и возможностях противника |
| **OSINT (открытые источники)** | Журналистика, OSINT-аналитики, верификация новостей и соцсетей |
| **Кибербезопасность** | Оценка достоверности индикаторов компрометации (IOC), отчётов об угрозах |
| **Научные и экспертные отчёты** | Где нужно явно передать уровень уверенности в утверждениях |

В контексте **инвестиционной и геополитической аналитики** Admiralty Code применяется для взвешивания доказательств из новостей, отчётов think tank и рыночных данных: каждому «evidence» присваивается рейтинг (например B2), и агрегация гипотез учитывает эти веса.

## 5. Основные компоненты методики

### 5.1. Надёжность источника (Source Reliability) — буквы A–F

Оценка даётся на основе **истории** источника (для HUMINT) или **технической оценки** его возможностей (для технических средств). Критерии: подлинность (authenticity), доверие (trustworthiness), компетентность (competency).

| Код | Значение | Критерий (по UK JWP 2-00 / NATO) |
|-----|----------|----------------------------------|
| **A** | Completely reliable | Нет сомнений в подлинности, доверии или компетентности; история полной надёжности |
| **B** | Usually reliable | Небольшие сомнения; в прошлом в большинстве случаев давал верную информацию |
| **C** | Fairly reliable | Есть сомнения, но в прошлом предоставлял верную информацию |
| **D** | Not usually reliable | Существенные сомнения, но иногда давал верную информацию |
| **E** | Unreliable | Ненадёжен; история неверной информации |
| **F** | Reliability cannot be judged | Нет основы для оценки (новый или неизвестный источник) |

### 5.2. Достоверность информации (Information Credibility / Accuracy) — цифры 1–6

Оценка даётся **по каждому сообщению отдельно**: насколько оно подтверждено, логично и согласуется с другими данными.

| Код | Значение | Критерий |
|-----|----------|----------|
| **1** | Confirmed | Подтверждено другими независимыми источниками; логично само по себе; согласуется с другой информацией по теме |
| **2** | Probably true | Не подтверждено; логично; согласуется с другой информацией |
| **3** | Possibly true | Не подтверждено; достаточно логично; согласуется с частью другой информации |
| **4** | Doubtful | Не подтверждено; возможно, но нелогично; другой информации по теме нет |
| **5** | Improbable | Не подтверждено; нелогично; противоречит другой информации |
| **6** | Truth cannot be judged | Нет основы для оценки достоверности |

### 5.3. Комбинированная оценка и отчётность

- Итоговая градация — **два символа**: например **A1** (наивысшая уверенность), **F6** (низшая).
- В отчёте **оценку получает каждое использованное доказательство**, а не только итоговый вывод. Итоговый вывод формулируется с оговорками (caveats), отражающими уровень уверенности по кодам.
- В **intelligence cycle** оценка по Admiralty выполняется на этапе **обработки (processing)**: признаётся, что собранные данные нельзя принимать «как есть».

## 6. Критика и ограничения

- **Субъективность:** и надёжность источника, и достоверность информации зависят от суждения аналитика; разные аналитики могут присвоить разные коды.
- **Статичность:** рейтинг источника может не учитывать изменение качества со временем (источник «испортился» или, наоборот, стал точнее).
- **Нет явной числовой шкалы:** для машинной агрегации (как в Aladdin) коды нужно маппить на веса (например A=1.0, B=0.8, …, 1=1.0, 2=0.8, …) по соглашению.
- **Credibility 6 и Reliability F:** «нельзя оценить» создаёт неоднозначность при агрегации — в части методик их исключают или дают нейтральный вес (0.5).

Несмотря на критику, система остаётся стандартом де-факто в NATO и Five Eyes и даёт общий язык для передачи уверенности между аналитиками и потребителями разведки.

## 7. Применение в Aladdin (кратко)

- Каждой записи в **fact_evidence** присваивается рейтинг по Admiralty (например, поле `source_reliability` = 'B', `info_credibility` = 2).
- В конфиге **config/source_reliability.yaml** задаётся маппинг source_id (bbc, ap, tass, …) на типовой уровень надёжности источника.
- **Достоверность** обновляется по правилам: при подтверждении из ≥2 независимых источников — повышение до 1; при противоречии с другими фактами — понижение.
- При расчёте **score гипотезы** используется взвешивание доказательств: `admiralty_weight = f(reliability) × f(credibility)`; evidence с A1 весит больше, чем D4.

**Admiralty Code и защита от дублей:**

Admiralty оценивает качество **конкретного** evidence (надёжность источника и достоверность сообщения), но **не заменяет** проверку независимости источников и кластеризацию событий (event dedup). Множество перепечаток одного agency wire не должно повышать `score_support` — это задача слоя event clustering (см. PREDICTIVE.md, §13.1c). Admiralty-рейтинг `info_credibility` повышается до 1 только при подтверждении из **независимых** источников (разные редакции, разные типы каналов), а не из перепечаток одного исходного сообщения.

Подробности формул и таблиц — в **[../PREDICTIVE.md](../PREDICTIVE.md)**, разделы 10.1, 13.1–13.2.
