# Универсальная диагностика API-провайдеров

Паттерн, применённый к Gemini Vision (3 июня) и Groq STT (22 июня). Работает для любого недоступного API.

## Алгоритм диагностики (строго по порядку)

| Шаг | Что проверить | Команда | Критерий |
|-----|---------------|---------|----------|
| 1 | VPN жив? | `sudo awg show` | `latest handshake: N seconds ago` |
| 2 | Трафик идёт через VPN? | `curl -s ifconfig.me` | Должен быть IP VPN-сервера (для диапазонов в AllowedIPs) |
| 3 | TCP/TLS до endpoint? | `curl -sv --max-time 5 https://api.example.com` (verbose) | TLS handshake OK |
| 4 | API отвечает? | `curl` с ключом и минимальным payload | Осмысленный HTTP-ответ (не таймаут) |
| 5 | Без VPN? | Убрать диапазон из VPN-маршрутов, повторить шаг 4 | Сравнить коды ошибок |
| 6 | Другая модель/эндпоинт? | Gemini: 2.0 vs 2.5 flash. Groq: `/models` vs `/chat` | Разные квоты/доступ |

## Интерпретация кодов

| Код | Значение | Действие |
|-----|----------|----------|
| 200 | Всё работает | — |
| 403 plain | Ключ невалидный или аккаунт не активирован | Проверить консоль провайдера |
| 403 + `error code: 1010` | Cloudflare WAF банит IP | Сменить VPN-сервер или IP |
| 429 | Квота (RPM или RPD) | Ждать или переключить модель |
| Таймаут | Сеть/VPN | Проверить маршруты, `traceroute` |

## Pitfalls

1. **Не предполагай причину без проверки.** «IP-бан», «DPI», «кавычка в ключе» — все три гипотезы оказались ложными в разное время. Проверяй.
2. **Разные модели = разные квоты.** Gemini 2.0 и 2.5 flash имеют раздельные лимиты. Groq может иметь раздельный доступ для STT и Chat API.
3. **Cloudflare WAF ≠ бан API-ключа.** Код 1010 означает блокировку на уровне CDN, а не на уровне провайдера.
4. **Перед предложением «поставить локально» — проверь ресурсы.** На 2 ядрах Xeon 2011 года без GPU даже tiny-модель faster-whisper будет работать неприемлемо медленно. Всегда: `nproc`, `free -h`, `lspci | grep -i vga`.
5. **Никогда не вставляй API-ключи в systemd override через `Environment=`.** Ключ в `env.conf` виден всем, у кого есть доступ к systemd. Вместо этого: `EnvironmentFile=/home/mia/Mia/.env` в systemd override — тогда все ключи из .env попадают в окружение процесса безопасно (файл .env защищён правами 0600). Паттерн случался дважды: GOOGLE_API_KEY (3 июня) и OPENAI_API_KEY (22 июня). Больше не повторять.
